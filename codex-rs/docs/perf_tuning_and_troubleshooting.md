# 性能调优实践与排障手册

## 目标与方法
- 目标：在保证正确性的前提下提升延迟与吞吐，降低内存与 IO 开销。
- 方法：度量—定位—优化—回归验证 的闭环，用 serena-bench 做基准测试。

## 关键开关与建议
- 日志级别：生产建议 info；排障时临时提升到 debug/trace。
- 工具并发：根据 CPU/IO 资源设置并发与队列长度，避免爆发式抖动。
- 超时与重试：为外部工具/HTTP 设置合理 timeout_ms 与重试退避。
- 缓存与DB：热点数据落盘/内存缓存，减少重复调用。
- 流水线分段：将复杂任务拆分阶段并监控每段耗时。

## 度量与可观测性
- 指标：请求耗时、错误率、工具调用数、并发度、缓存命中率。
- 日志：结构化输出（JSON）便于检索；关键路径加埋点。
- 追踪：启用分布式追踪（如 trace-id）贯通 CLI/LSP/Agent。

## 常见瓶颈与优化思路
1. 工具调用慢：
   - 启用连接复用（HTTP keep-alive），合并小请求。
   - 将热点命令改为本地 process 工具或直接库调用。
2. LSP 响应抖动：
   - 限制同时解析文件数；缓存增量索引。
3. 代理策略低效：
   - 简化规划（sequential/批量），避免过度反复。

## 排障流程
1. 复现：记录输入、配置、版本与时间戳。
2. 切分：区分 CLI/LSP/Agent 入口问题，缩小到 Tools/DB/网络层。
3. 证据：抓取 debug 日志、指标曲线、外部依赖状态。
4. 对策：回滚配置、降级功能或启用旁路；提交最小复现样例。

## 基准与回归
- 使用 serena-bench 设立标准用例；每个优化提交都跑一次。
- 记录历史结果，预警显著回退。
