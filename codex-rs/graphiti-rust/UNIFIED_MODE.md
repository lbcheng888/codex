# Graphiti 统一智能模式

## 为什么选择统一模式？

传统的多模式设计让用户困惑：
- 需要手动选择使用哪种模式
- 不同模式功能差异大
- 配置复杂，难以优化

**统一智能模式**自动选择最佳方案：
- 有 LLM？用它做高级提取
- LLM 失败？自动降级到规则 NER
- 只想快速？直接用嵌入搜索

## 架构设计

```
┌─────────────────────────────────────────┐
│           用户输入文本                    │
└────────────────┬────────────────────────┘
                 │
     ┌───────────▼───────────┐
     │    智能路由器          │
     │ (自动选择最佳方案)     │
     └───────────┬───────────┘
                 │
    ┌────────────┼────────────┐
    │            │            │
┌───▼───┐   ┌───▼───┐   ┌───▼───┐
│  LLM  │   │  NER  │   │ 嵌入  │
│(高级) │   │(规则) │   │(快速) │
└───┬───┘   └───┬───┘   └───┬───┘
    │            │            │
    └────────────┼────────────┘
                 │
         ┌───────▼───────┐
         │   统一输出     │
         │ (实体+关系+嵌入)│
         └───────────────┘
```

## 智能降级策略

1. **LLM 优先**（如果可用）
   - 提取复杂实体和关系
   - 生成智能摘要
   - 上下文理解

2. **规则 NER 备用**
   - LLM 不可用时自动启用
   - 支持中英文实体识别
   - 基于邻近度推断关系

3. **嵌入始终运行**
   - 保证语义搜索功能
   - 向量相似度匹配
   - 跨语言理解

## 使用示例

```bash
# 只需一个配置文件
./graphiti-mcp-server --config config.unified.toml

# 系统会自动：
# 1. 尝试连接 Ollama（如果配置）
# 2. 如失败，使用规则 NER
# 3. 始终生成 Qwen3 嵌入
```

## 配置说明

```toml
[graphiti]
mode = "auto"              # 自动选择最佳方案
extraction_strategy = "progressive"  # 渐进式提取

# 功能全部启用，系统自动判断
enable_deduplication = true
enable_relationship_inference = true
enable_temporal_analysis = true
```

## 优势对比

| 特性 | 传统多模式 | 统一智能模式 |
|-----|----------|------------|
| 配置复杂度 | 高（3个配置文件） | 低（1个配置文件） |
| 用户决策 | 需要选择模式 | 自动优化 |
| 容错能力 | 低（单点失败） | 高（自动降级） |
| 性能 | 固定 | 自适应 |
| 扩展性 | 需要新模式 | 自动集成 |

## 技术实现

```rust
// 智能实体提取
match &self.llm_client {
    Some(llm) => {
        // 尝试 LLM
        match llm.extract(...).await {
            Ok(result) => use_llm_result(),
            Err(_) => fallback_to_ner()  // 自动降级
        }
    },
    None => use_ner_directly()  // 直接用 NER
}
```
